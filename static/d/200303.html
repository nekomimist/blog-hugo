Mar.1,2003 (Sat)

■「パティシエなにゃんこ」(ぱじゃまソフト) [GAME]

+ まぁ、1ヶ月後にDVD版が出るのだが、DVD版にはサントラはつかないって事なので買ってみた。

+ …SafeDisc2.xかぁ。起動毎にチェックするのはどーもウザい。SafeDisc自体の賛否はさておくとしても、やるならRUNEのゲームのように忘れたころにCDチェックが入る仕様にしてほしいもんだ。

+ 1stプレイはパティシエのみちるさん。乳に惹かれたのかと問いつめられたら否定できん(ぉ 途中のデートと終盤以外強い印象がなかったかも…。最後はお約束といえばお約束だけど、俺はこういうお約束展開は嫌いじゃないのでまぁよし。

+ 2ndプレイは幼なじみのかなで。その髪型と好き好き電波の発生ぶりが某あかり似 (^^; しかし、幼なじみものの王道的展開が実にこっぱずかし〜♥←うれしいらしい

+ なんとなく乳の大きい人から倒していってるような気がしたあたりで寝る。

Mar.2,2003 (Sun)

■「パティシエなにゃんこ」(ぱじゃまソフト) [GAME]

+ 3rdプレイは対面のケーキ屋ショコラ・ル・オールのお嬢様の冬華狙いで。ああ、しかし、最近俺がやるゲームで海原エレナが声アテてるキャラは子供っぽい系統ばっかやな…。

+ …ありゃ、亜里咲に行ってしまった。ヒロインは4人(ミオ・かなで・冬華・みちる)と認識してたんだけど、亜里咲ちゃんエンドもあるんだなぁ。妹の茉理のエンドもあるのかな?

+ 分岐くさいところに戻って冬華へ。あーもー、なんつーか、ご都合主義だろうがなんだろうが、いいね。かなでがちょいと不憫だけど、まあ、あの立場にいる幼なじみとしては宿命だし :-)

+ 4thは猫魔法使いのミオへ。なるだろうと思った方向の話そのままだったが、それでいいんだよな、うんうん。しかしやはりかなでは不憫だった…。

+ で、CGも回想も全部埋まって終了。おもしろかった。俺的評価は8点で佳作かな。

Mar.3,2003 (Mon)

■「Routes」(Leaf) [GAME]

+ 「パティシエなにゃんこ」が終わったので始めようかと思ったら、なかなかタイトル画面に辿りつけなくて苦労した (^^;

+ HIMIKOの通常使用状態ではSafeDisc2のチェック後、ムービーに移るところでほぼ100%死亡。BIOS設定でHyper-ThreadingをOFFにするか、Hyper-ThreadingはON/BGMをOFFで起動するかするとムービー再生に辿りつけてその後タイトル画面に行ける模様。

+ 後者で対処した場合、開始した後にBGMをONにしてもOKだけど、エンディングに入るところでもう一度死亡したりする(*1)。しかしBGM設定を切るとエンディング曲が流れない罠 :-)

+ ちなみに1stプレイはリサでした。やぁ、なんかけっこうよかったですよ?

+ 次にいきたい気分になったけど明日にしよう (^^;

■cmail→wanderlust [COMP]

+ こんな時間(28:00)にMUAの移行作業してる俺って会社勤め社会人としてどーよ(笑)

+ ま、cmailはかれこれ6年ほど使ってて馴れてはいるんだけど、IMAP4対応って点ではちと弱いし、最近動きがないのでWanderlustに乗りかえてみた。

+ 1996年頃にMewからcmailにのりかえたときは、過去のメールの変換に苦労したもんだけど、今は過去のメールもほぼ全てIMAP4サーバのAKANE上にあるので、WanderlustだろうがcmailだろうがMewだろうがGnusだろうがWindowsびしばしなMUAだろうが、IMAP4対応でありさえすればさほど苦労なく乗りかえられていいやね。

*1: 1回中1回なので再現性あるかどうかは不明。

■ノートPCを買いたい [COMP]

+ 現行主力ノート機のMACHIKAはLOOX T8/80Wで、さすがに非力さが目立つような感じ。TM5800-800MHzでSDRAM(*1)。でRAGE Mobiliy-Mだもんなぁ。

+ しかし俺の物欲をくすぐる現行機種はほとんどない。LOOX T8/80Wみたいな2スピンドルで1.5Kg・10.6型1280x768液晶なんて妙な構成のノートに馴れちゃうと、他社の2スピンドルノート機は2Kg級で重くてデカいし、サブノート機は液晶のドット数が減っちゃうし。かといってLOOX Tの最新機種はTM5800-933MHzでSDRAMでMobility RADEONなので大差ない感じだし。

+ やはりLOOX Tの後継機待ちか。Pentium-MになるのかASTRO(TM8000)になるのか、そもそも出るのかもわからんが :-)

■自動置換フィルタと戯れる [nDiary]

+ autoreplace.rbを使おうってことでふにふに作業する。

+ どうもautoreplace.txtの最初の要素に「regexp内で特別な意味を持つ文字」が入ってた時に妙な動作になる。こんな↓感じでよいのかな。

--- autoreplace.rb.orig 2003-01-23 07:01:06.000000000 +0900
+++ autoreplace.rb      2003-03-30 04:42:57.000000000 +0900
@@ -22,7 +22,7 @@
               if %r(^/(.+)/[ioxm]?$) =~ str then
                 regexp = eval(str)
               else
-                regexp = Regexp::compile(str)
+                regexp = Regexp::compile(Regexp::quote(str))
               end
               @autoreplacelist << [regexp, replace.chomp]
             end
*1: CMSの原理を考えるとRAM速度は速ければ速いほどよいはず。
