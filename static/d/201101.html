■調理用具をいろいろいれかえた話 [MISC]

+ ボロくなってきていた片手鍋とフライパンを買った。鍋はコレでフライパンがコレ。安物でもなく高すぎでもないあたりを狙ってみた。

+ フっ素樹脂コートじゃないフライパンは、油まわししたり温度気にしたりしないといけないあたりが大変だが、これはこれで面白いので良しとする。ただの鉄フライパンよりは扱いが楽だしね。鍋はピカピカすぎてちょっと困りものかもしれん(笑)

+ ついでに包丁をちゃんと研いでみたらやたら切れるようになった。もともと今のマンションを買った時に親から中古で貰ったものなので、買いかえなきゃダメかなと思っていたが、これでしばらくは買いかえなくてよいかもしれない。……つーか、マメに研ごう。

+ 次は両手鍋をどうすかだな。20cmの安物と22cmのホーローのがあるのだが、安物はいい感じにボロくなってきているし、ホーローのは重いしメンテが大変なので、片手鍋と同じジオプロダクトの22cm鍋か20cmポトフ鍋でも買って両方処分するのもよいかもなー。

Jan.22,2011 (Sat)

■FIENAさんの状況 [COMP]

+ P8P67 DELUXEのBIOS1003だとクロック倍率を44倍にすると一切起動しなくなる状況だったので、えいやっとBIOS1053βに入れかえてみた。とりあえず深く考えずに46倍にしたら、5103×46=4.74GHzで動いた。電圧AUTO設定だとかなり熱くなるので、もうちょっとチューニングが必要だな。

+ P8P67 DELUUEのP67側のSATAにPIONEER DVR-S16Jを継ぎぎ、かつ起動するWindows7にIntel Rapid Storage Technology 10.1をインストールしているとWindows7が起動中にはまるので(*1)、オンボードのMarvell 88E9128にDVR-S16Jをつないでいる。しかし、この状況でVMware Workstation内のWindowsXPのCDドライブをこのDVR-S16Jに向けると、仮想マシン内だけでなくホストのWindows7側でも動画の音声がブツブツ切れる問題が出る。DPC latency Checkerを動かすと真っ赤。仮想マシン側のオートランが動いたままなのが悪いのだろうが、それにしたってMarvell 88E9128のドライバのデキが悪いのも事実だろうな。

■KONAさん実体交換計画 [COMP]

+ 無事FIENAの中身がCore i7 2600K+P8P67 DELUXEになったので、KONAさんをML115G1から、余剰パーツで組んだマシンにする。

+ CPUとM/Bは旧FIENAのIntel Core2Extreme QX9650+Rampage Formula+DDR2-800 2GB×4を使う。CPUクーラーは手持ちに未開封のXIGMATEK Dark Knight S1283Vがあったのでそれを使った。電源はFIENA用にKRPW-P800W/85+を買って、余ったKRPW-J600Wを移植した。グラフィックカードもない(*2))のでRADEON HD5450を買った。DVDドライブもないのだが、FreeBSD運用には不要なので買わない。そんなところでハード的には準備完了。

+ あとは旧KONAからHDD6台を移植すればいいのだが、ZFSな2TB×4のpoolはいいとして、/がnVIDIAのソフトRAID1(80GB×2)でイマイチなんだよなー。むー、どうしたものやら。

*1: IRST 10.1の問題らしい。
*2: 厳密にはRADEON HD4870があるけど、これをサーバ用に使うのは電気代が(笑
