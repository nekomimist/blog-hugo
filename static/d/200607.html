Jul.1,2006 (Sat)

■AKANEの今後 [COMP]

+ AKANE(FreeBSD6.1R)の維持が面倒臭くなってきた。AKANEが今やっている事は以下のことだが、実のところどれもこれも俺専用のサービスなんで、何か割り切って楽になれないかな?

+ まずはsamba利用のファイルサーバ。しかしファイルはHIMIKO(メインのデスクトップPC)から使うのがほとんどなんだからHIMIKOに蓄積するほうがよさそう。幸いHIMIKOのM/BはSATAでRAID1できるみたいだし、300〜400GB級のHDDを3台買って2台でRAID1して1台スペアにすると幸せかな?

+ 次にimap4サービスの提供。fetchmailでメールを取ってきてcyrus-imapdにメールを蓄積している。これはなくなると痛いので必須。

+ あとは外部へのsshサービスの提供。会社や実家からsshでAKANEにログインして、会社からscpしたり、Emacsの上でwanderlustでメール読んだり、navi2chで2ch見たり、portforwardで外からHIMIKOのリモートデスクトップやimap4アクセスしたりする。これもなくなると痛すぎなので必須。

+ 以上。つまりimap4とsshだけじゃん。メールの方はfetchmailとcyrus-imap4とスパム避け(今はSpamAssassinとbsfilter)とウィルススキャン(今はclamav)が動けばいい。sshの方は、sshとscreenとemacs一式が動けばあとはなんとでもなる。

+ ……MacOS Xでなんとかなってしまう気がしてきた。Mac mini買って試してみるかな?

Jul.8,2006 (Sat)

■Mac mini雑感 [COMP]

+ 届いたので開ける。箱はiPodやAirMacとテイストが同じで実に「らしい」。

+ 初めてMacに触れて「肌に合わねー」と思ったのは20年くらい前だっけ……とか思いつつ起動。くるくる回るマウルカーソルを見て学生時代に何度か使ったNeXTSTEPを思い出した。初代NeXT Cube+Dimensionボードの構成で、何かやるたびにすぐくるくる回って待たされたっけ。

+ ……さて、とりあえずどこで何設定すればいいのかわからんなー。

+ いろいろつれづれなるままに。

  • 手元に置くとすると音が気になるな。遠くに置けばいいけど。
  • メニューが画面の一番上に出るのは馴れないなー。
  • Safariは画面が出るまでがFirefoxより遅い。なんだろう。
  • Firefoxの体感速度がHIMIKOのそれよりあきらかに遅い。ま、CoreDuo(1.66GHz)とAtholn64X2 4600+(2.4GHz)の差とか、GeForce7600GS OC版とi945Express内蔵グラフィックの差か……。
  • iTunesはMacでもやっぱ重いんだね :-)
  • iTunesはタグのエンコードにすごくシビア。Windows版iTunesが何でも食うのにー。
  • ことえり馴れねー。あとでAquaSKKを入れよう。
  • Carbon Emacsを入れてみた。Meadow 3.00と同じで最近のEmacs CVS headベースなので、特に違和感はない。フォント設定をいろいろ追いこまないとな。
  • ターミナルを開いてディレクトリ構成を眺めると……旧い世界と新しい世界がグチャっとした感じが。
  • DarwinPortsは馴れ親しんだFreeBSDのportsと同じ感じで使えてよいね。しかしIntel Mac非対応のモノがけっこうあってアレかなー。

+ さて、AKANEの役割を代替するまでに持っていけますかな?

Jul.12,2006 (Wed)

■Mac miniでBootcampでFreeBSDその2 [COMP]

+ さて、昨日の続き。ぽいぽいとFreeBSD 6-STABLEのCDIMAGEから起動して、MacOS Xからリサイズして後半60GBほどを開けておいた内蔵ディスク後半を使ってインストールしつつ再起動すると普通に起動する(*1)。

+ んで、Marvellから落としてきたif_myk.koをkldloadしたら普通に成功。勝利の予感。じゃあってことで、loader.confにふにふにif_myk_load="YES"など書いてネットワーク設定をぽいぽいと行ってから再起動すると……イケてますぜ、旦那。

+ ま、とりあえず必須portsをふにふにと仕込んどこう。本気で設定するのは週末だなー。

*1: ちなみにMBRには標準MBRを入れた。bootmgrでどうなるかはシラネ。

Jul.14,2006 (Fri)

■「Schoolぷろじぇくと☆」(アトリエかぐや) [GAME]

+ 遥→蛍→小桃→雫→すみれ先生で終了。

+ "コミカルな学園モノ"をやりたかったんだろうが、結果としてエロ重視でもなく、ギャグとしては面白くないツッコミ所が多いだけの平凡なゲームに仕上がってしまった感じがするねー。

+ ヒロイン視点に移ることがわりと多くて、雫あたりではうまく使われてもいたのだが、視点が変わったことがわかりづらいので一瞬わけがわからなくなる。メッセージウィンドウが変わるくらいはやったほうが親切だったかも。

+ キャラのイチオシは雫(CV:茶谷やすら)。あまりに"いかにもなツンデレ"すぎではあるのだれけど、ヒロイン視点がうまく効いていて、わりと上手く描かれていたと思う。話のオチはアレだが小桃や蛍よりは相当マシだし(笑)

+ 評価は6点で「普通」ってとこか。

■Mac miniでBootcampでFreeBSDその3 [COMP]
